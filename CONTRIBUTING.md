# 🤝 コントリビューションガイド

IT Engineer Knowledge Architectureプロジェクトへのご参加ありがとうございます！このガイドでは、プロジェクトに貢献する方法について説明します。

## 🎯 プロジェクトのミッション

ITエンジニア（特にインフラ系）が実務で必要とする知識を体系化し、段階的な学習ロードマップを提供することで、効率的なスキル向上と実務での即戦力化を実現する。

## 🚀 参加方法

### 1. 💬 議論・質問・アイデア共有

**GitHub Discussions**で以下のようなトピックを歓迎します：

- 学習体験の共有
- 実務での活用事例
- 新しい書籍アイデア
- 学習パスの改善提案
- 技術トレンドに関する議論

### 2. 📝 Issues作成

以下のテンプレートを使用してIssueを作成してください：

- **📚 書籍企画提案**: 新しい書籍の企画や既存書籍の改善
- **🗺️ 学習パス改善提案**: 学習順序や難易度の調整提案
- **⚡ 品質改善提案**: 内容の誤りや改善点の報告

### 3. 🛠️ Pull Request作成

ドキュメントの改善、新しい学習パスの提案、誤字脱字の修正などをPull Requestで送ってください。

## 📋 コントリビューション種別

### 🔴 高優先度（すぐに貢献できる）

#### ドキュメント改善
- 誤字・脱字の修正
- リンク切れの修正
- 表記統一（である調、専門用語など）
- 説明の分かりやすさ向上

#### 学習体験の共有
- 実際に書籍を学習した体験レポート
- 実務での活用事例
- つまずきポイントと解決方法
- 学習時間・難易度の実感値

#### 品質向上提案
- 技術的な誤り・古い情報の指摘
- より良い実践方法の提案
- セキュリティ・パフォーマンスの改善案

### 🟡 中優先度（専門知識が必要）

#### 学習パス設計改善
- 書籍間の学習順序最適化
- 前提知識の調整
- 専門分野別パスの改善
- 難易度カーブの調整

#### 新規コンテンツ提案
- 実践的な演習課題
- ケーススタディの追加
- チェックリスト・テンプレート
- 補足解説・FAQ

### 🟢 低優先度（長期的な貢献）

#### 書籍企画・執筆
- 計画書籍7冊の執筆協力
- 新しい書籍企画の提案
- 技術監修・レビュー
- 翻訳・国際化

#### システム・自動化
- GitHub Actions改善
- 学習進捗管理システム
- 品質チェック自動化
- CI/CD改善

## 🎯 品質基準

### 📝 文書作成基準

#### 文体・表記
- **文体**: である調で統一
- **専門用語**: 初出時に説明を併記
- **表記統一**: アーキテクチャ（アーキテクチャーではない）
- **改行**: 読みやすさを考慮した適切な改行

#### 構造・内容
- **階層構造**: 見出しレベルの適切な使用
- **実践的内容**: 具体例・サンプルコードの充実
- **検証可能性**: 記載内容の事実確認・検証可能性
- **更新性**: 技術変化に対応できる記述

### 🔍 技術的正確性

#### 必須確認事項
- **最新性**: 記載する技術・ツールのバージョン確認
- **動作確認**: コマンド・手順の実際の動作確認
- **環境依存**: 動作環境・前提条件の明記
- **セキュリティ**: セキュリティベストプラクティスの確認

#### 推奨事項
- **複数環境での検証**: 異なるOS・バージョンでの確認
- **エラーケース**: よくあるエラーと対処法の記載
- **参考情報**: 公式ドキュメント・信頼できる情報源の併記

## 📊 レビュープロセス

### 1. 自己チェック

#### 提出前の確認事項
- [ ] 文体・表記統一の確認
- [ ] 技術的内容の事実確認
- [ ] リンク・参照の正確性
- [ ] 他のコンテンツとの整合性

### 2. コミュニティレビュー

#### レビュー観点
- **内容の正確性**: 技術的な誤りがないか
- **分かりやすさ**: 対象読者にとって理解しやすいか
- **実用性**: 実務で活用できる内容か
- **体系性**: 全体構造との整合性があるか

#### レビュー期間
- **軽微な修正**: 3-5日
- **内容の追加・変更**: 1-2週間
- **大規模な変更**: 2-4週間

### 3. メンテナー承認

最終的にプロジェクトメンテナーが承認し、mainブランチにマージされます。

## 🌟 貢献者認定

### 🏆 コントリビューター認定レベル

#### 🥉 Bronze Contributor
- **条件**: 1つ以上のPull Requestがマージ
- **特典**: README.mdのContributors欄に掲載

#### 🥈 Silver Contributor
- **条件**: 5つ以上の意義あるコントリビューション
- **特典**: 専用バッジ + Discord特別チャンネルへの招待

#### 🥇 Gold Contributor
- **条件**: 継続的な高品質コントリビューション（10件以上）
- **特典**: 技術監修者としての認定 + 執筆協力の打診

#### 💎 Platinum Contributor
- **条件**: プロジェクトの発展に大きく貢献
- **特典**: プロジェクト運営メンバーとしての招待

### 🎖️ 特別貢献認定

- **📚 Author**: 書籍執筆・共著
- **🔍 Technical Reviewer**: 技術監修・査読
- **🌐 Community Leader**: コミュニティ運営・イベント企画
- **🛠️ Tech Lead**: 技術基盤・自動化の構築

## 📞 コミュニケーション

### 🗣️ コミュニケーションチャンネル

#### GitHub
- **Issues**: 具体的な問題・提案
- **Discussions**: 議論・質問・アイデア共有
- **Pull Requests**: コード・ドキュメント改善

#### メール
- **プロジェクト運営**: knowledge@itdo.jp
- **執筆協力**: 個別相談・企業連携

### 💬 コミュニケーションガイドライン

#### 推奨事項
- **建設的なフィードバック**: 改善案を含む具体的な指摘
- **丁寧な表現**: 相手を尊重した言葉遣い
- **事実に基づく議論**: 主観よりも客観的な根拠を重視
- **学習者視点**: 常に学習者の利益を最優先

#### 禁止事項
- **人格攻撃・誹謗中傷**: 個人への攻撃的な発言
- **スパム・宣伝**: 無関係な宣伝・営業活動
- **著作権侵害**: 他者の著作物の無断使用
- **ハラスメント**: 性別・年齢・経験等による差別的発言
