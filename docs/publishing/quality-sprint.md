# 品質スプリント運用

## 目的

シリーズ内書籍の品質（誤字脱字、表記ゆれ、軽微な構造不整合、外部リンク棚卸し、内容の陳腐化リスク）を、定期的な小粒PRで継続的に低減します。

## 実施頻度

- 目安: 隔週（1日/15日）または月次
- `Quality Sprint Reminder` ワークフローが、スプリント用Issueを自動起票します（`[Quality Sprint] YYYY-MM-DD`）。

## 運用ルール（推奨）

- 1回のスプリントで対象は **2〜4冊**（過負荷を避け、レビュー品質を維持）
- 1PRは **1カテゴリ**（例: 外部リンク棚卸し / 誤字脱字 / ファクトチェック / 図表レイアウト）
- 公式仕様/EOL/非推奨の修正は、根拠（公式ドキュメント等）を併記
- PRは Book QA を通過し、GitHub Pagesで表示確認（PC/モバイル）を行う

## 進め方（概要）

1. 自動起票されたスプリントIssueで対象書籍を選定
2. 対象書籍ごとに小粒PRを作成し、Issue側チェックボックスを消し込み
3. スプリントIssueをクローズ（次回へ繰り越しがあれば明記）
